## The Problem

India has **12 million kirana stores**. Each owner wakes up not knowing which products are running out, which shelves are disorganised, or what their neighbourhood is short on. They have no scanner, no ERP, no inventory system. They have a phone and WhatsApp.

ShelfScan AI gives every kirana owner a **personal AI retail analyst in their pocket** вҖ” no app download, no training required. They photograph their shelf. We send back a Hindi voice note telling them exactly what to order, how much, and why.

## Features

### рҹ§  3-Agent AI Debate System
The core innovation. Not one AI opinion вҖ” a structured debate between three specialist agents, each with a different perspective:

| Agent | Model | Role |
|-------|-------|------|
| **Presenter** | Gemini 1.5 Pro | Builds the most comprehensive restock plan вҖ” urgency, quantities, revenue at risk |
| **Critic** | GPT-4o Mini вҶ’ Groq LLaMA-3.1 вҶ’ Gemini Flash | Stress-tests the plan against real cash flow constraints (kirana daily revenue: вӮ№2KвҖ“вӮ№10K) |
| **Decider** | Gemini 1.5 Pro | Synthesises the debate into a final ranked action plan, then writes the Hindi voice note |

The Critic has a **priority fallback chain** вҖ” it tries GPT-4o Mini first, then free Groq, then free Together AI, then Gemini Flash. The system works even if you have zero paid API keys.

### рҹ‘ҒпёҸ Gemini Vision Analysis
Every product on the shelf is detected and characterised:
- **Name & brand** вҖ” read directly from labels
- **Stock level** вҖ” `critical` / `low` / `ok` / `overstocked`
- **Shelf position** вҖ” `top` / `middle` / `bottom` / `eye_level`
- **Facing** вҖ” is the label pointing toward the customer?
- **Overall health score** вҖ” 0 to 100

### рҹҺҷпёҸ Hindi Voice Notes
ElevenLabs `eleven_multilingual_v2` model generates natural Hindi/Hinglish audio вҖ” warm, conversational, like a knowledgeable friend calling you. Uploaded to Cloudinary CDN, delivered via WhatsApp.

### рҹ“Қ Neighbourhood Intelligence
Demand data is aggregated by pincode nightly. Owners see:
- What products are running out across their neighbourhood
- Which categories are in high demand locally
- How their shelf health compares to nearby stores

### рҹ“ұ WhatsApp-First Interface
No app download. No login. Just WhatsApp вҖ” the interface 500M+ Indians already use every day. The full AI pipeline triggers from a single photo message.

### рҹҢҗ Web Dashboard
A full single-file dashboard (`shelfscan-v3.html`) for owners who prefer a browser interface вҖ” scan history, inventory tracking, debate logs, voice note archive, neighbourhood map.

## Getting Started

### Prerequisites
- Python 3.11+
- A Supabase account (free tier works)
- Gemini API key (Google AI Studio вҖ” free)

### 1. Database Setup

```bash
# Open your Supabase project's SQL Editor and run schema.sql
# Creates: stores, scans, detected_products, debate_rounds,
#          voice_notes, product_catalog, neighborhood_demand, whatsapp_messages
```

### 2. Install & Run

```bash
git clone https://github.com/your-username/shelfscan-ai
cd shelfscan-ai

pip install -r requirements.txt
python main.py
# вҶ’ http://localhost:8000/docs
```

### 3. Open the Dashboard

```bash
open shelfscan-v3.html
# No server required вҖ” it's a single HTML file
```

## API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/` | Health check |
| `GET` | `/health` | Status + timestamp |
| `POST` | `/api/stores/register` | Register new kirana store |
| `GET` | `/api/stores/login/{whatsapp}` | Login by WhatsApp number |
| `GET` | `/api/stores/{store_id}` | Get store details |
| `POST` | `/api/scan` | **Full AI pipeline** вҖ” image вҶ’ vision вҶ’ debate вҶ’ voice вҶ’ DB |
| `GET` | `/api/dashboard/{store_id}` | All dashboard data |
| `GET` | `/api/scans/{store_id}` | Scan history with debates + voice notes |
| `GET` | `/api/scan/{scan_id}/details` | Full scan detail |
| `GET` | `/api/neighborhood/{pincode}` | Pincode demand intelligence |
| `GET` | `/api/voice/{scan_id}` | Voice note for a scan |
| `POST` | `/webhook/whatsapp` | **Twilio WhatsApp webhook** |

## Database Schema

| Table | Purpose |
|-------|---------|
| `stores` | Registered kirana store owners |
| `scans` | Each shelf scan вҖ” image URL, health score, final recommendation |
| `detected_products` | Every product found in each scan |
| `debate_rounds` | All three agent outputs per scan |
| `voice_notes` | Hindi audio URL + text per scan |
| `product_catalog` | Global FMCG reference (seeded with 15 common products) |
| `neighborhood_demand` | Weekly pincode-level demand aggregation |
| `whatsapp_messages` | Full inbound/outbound WhatsApp message log |

All tables have Row Level Security enabled. Indexes on all foreign keys and frequently queried columns.
